# 网络代理说明

## 什么是代理？

### 简单比喻

想象您要给朋友送东西：
- **不使用代理**：您直接走到朋友家送东西
- **使用代理**：您把东西交给中间人，由中间人转交给朋友

在网络中，代理（Proxy）就是一个"中间人"，帮助您的电脑访问互联网。

---

## 为什么需要代理？

### 场景 1：公司/学校网络
- 公司或学校可能会限制访问某些网站
- 需要通过代理服务器才能访问外部网络

### 场景 2：地理位置限制
- 某些网站只在特定地区可用
- 使用代理可以模拟其他地区的访问

### 场景 3：安全与隐私
- 代理可以隐藏您的真实 IP 地址
- 保护您的隐私

### 场景 4：网络加速
- 某些代理服务器可以缓存内容，加快访问速度

## 如何判断是否需要代理？

### 方法 1：在浏览器中测试

1. 在浏览器中访问：https://api.mastergo.cn
2. 如果能访问，则不需要代理
3. 如果无法访问，说明可能需要代理

### 方法 2：询问 IT/网络管理员

如果您在公司或学校网络环境中，可以询问：
- "我需要访问 api.mastergo.cn，是否需要配置代理？"

### 方法 3：查看系统代理设置

**Windows**：
1. 控制面板 → Internet 选项 → 连接
2. 查看"局域网设置"中的代理配置

**Mac**：
1. 系统偏好设置 → 网络 → 高级 → 代理
2. 查看是否有已配置的代理

## 代理信息是什么格式？

### 代理地址的格式

```
协议://地址:端口
```

**示例**：
```
http://proxy.company.com:8080
https://proxy.company.com:8080
socks5://127.0.0.1:1080
```

### 各部分说明

| 部分 | 说明 | 示例 |
|------|------|------|
| 协议 | 代理协议类型 | `http`、`https`、`socks5` |
| 地址 | 代理服务器的地址 | `proxy.company.com`、`127.0.0.1` |
| 端口 | 代理服务器的端口 | `8080`、`1080` |

### 可能的代理信息示例

**示例 1**：
```
http://192.168.1.100:8080
```

**示例 2**：
```
https://proxy.example.com:3128
用户名: user
密码: pass123
```

**示例 3**：
```
socks5://127.0.0.1:1080
```
